title: UI大改修 - 単音/カラオケを統合した新UI設計

## 背景
- 現在のUIはMVP設計が中心で、機能追加に対する拡張性が不足している
- 単音モードとカラオケモードのUIが別画面として分かれており、モード切替の体験が断絶している
- 音程表示・カラオケバー・録音/再生の状態が画面ごとに分散している

## 目的
- 単音/カラオケを1つのメイン画面に統合し、モード切替を自然に行えるUIへ再設計
- 録音中/再生中の状態が一目で分かるステータス表現を統一
- 音程表示の可読性とリアルタイム性を強化

## 参考（現状デザイン）
- ファイル: `design/perfectPitch-ui.pen`
- 画面:
  - 単音: `PerfectPitch Main (New)`（id: `W6SAH`）
  - カラオケ再生: `PerfectPitch Karaoke`（id: `4Bmhs`）
  - カラオケ録音: `PerfectPitch Karaoke (Rec)`（id: `PTrOz`）
- 仕様メモ:
  - 録音中は正解バー（未来の音）を表示しない
  - 再生中のみ正解バーを表示
  - 単音画面に「録音中」ステータスを残す
  - 録音中は再生ボタン無効風（淡色）

## 変更方針（大改修の方向性）
- 画面構成の統合:
  - 単音/カラオケを同一画面内で切替
  - カード構成を整理し、共通部（ヘッダー/ステータス/操作）を固定
- 情報の優先順位:
  - 現在の音名・ドレミ・Hz・centsを最上段に集約
  - バー（単音 or カラオケ）を第2階層に配置
  - 再生/録音操作は画面下に集約
- 状態管理UIの統一:
  - ヘッダー右にモード切替（単音/カラオケ）
  - ヘッダー/カード内に「録音中/再生中」ステータス表示を統一
- 録音中カラオケ表示:
  - 未来バーは非表示
  - 直近履歴のみを薄いトレースとして可視化

## 具体タスク
- [ ] 新しいメイン画面のレイアウト草案作成（単音/カラオケ統合）
- [ ] モード切替UIと状態表示の設計統一
- [ ] カラオケバー（再生/録音）の表現を整理
- [ ] 既存カード構成（Live Notes/Playback）を再設計し統合
- [ ] 余白・タイポグラフィ・配色の再調整

## 受け入れ条件
- 単音/カラオケの切替が1画面内で完結する
- 録音中は未来バーが表示されない
- 録音中の再生ボタンは無効状態に見える
- 音程情報（音名/ドレミ/Hz/cents）が高い視認性で表示される
- 再生/録音操作の導線が分かりやすい

## 非目標
- 音声解析ロジックの変更
- API・データ構造の変更
